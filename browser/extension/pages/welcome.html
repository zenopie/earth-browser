Welcome to Earth Browser

Privacy-hardened browsing over the Reticulum network

What is Reticulum?

Reticulum is a cryptography-based networking stack that provides:

  • End-to-end encryption on every connection (X25519 + AES-256)
  • Initiator anonymity — servers cannot see your source address
  • Self-sovereign addressing — destinations are cryptographic keypairs, not domain names
  • Medium-agnostic transport — works over LoRa, packet radio, WiFi, TCP/IP, and more

Earth Browser routes your web traffic through Reticulum instead of the regular internet. No TLS certificates, no DNS, no IP addresses — just cryptographic destinations.

How It Works

You browse to .ret addresses like:

http://a4f2b8c91d3e7f06a4f2b8c91d3e7f06.ret/

Each address is a cryptographic destination hash. The proxy daemon (earthproxy) handles the SOCKS5 connection and establishes encrypted Reticulum links on your behalf.

Privacy Mode

How should Earth Browser manage your identity?

Your Reticulum identity is a cryptographic keypair. Privacy modes control when identities are created and destroyed.

Security Defaults

Earth Browser ships locked down. You choose what to unlock.

JavaScript Is Disabled

JavaScript is the primary vector for browser fingerprinting, data exfiltration, and code execution. It's off by default.

You can enable it per-destination or globally via the shield icon in the toolbar:

Green — JavaScript disabled. Maximum protection.
Yellow — JavaScript enabled for this specific destination.
Red — JavaScript enabled globally. Privacy degraded.

Always Blocked

These remain disabled regardless of JavaScript settings:

  • WebRTC (prevents IP leaks)
  • Service Workers & WebAssembly
  • Geolocation, Bluetooth, USB, Serial APIs
  • Requests from .ret pages to non-Reticulum destinations

All Tor Browser anti-fingerprinting patches remain active: canvas protection, letterboxing, font restriction, timing precision reduction.

You can change these settings anytime via the shield icon or the options page.
